Welcome to INST377!



This article is to help you get set up with a development environment that will let you test your own labs to see whether you're doing it right, before you submit your work. It's a separate article from the main lab assignments because some of you will have gotten into the course late, and will need a reference for setting up.

INST377 is a technical course about human problems. This means that, more than anything else, INST377 is about learning to be comfortable with new, potentially frustrating technical experiences. In this course, you will be given professional tools, which should help a lot with basic problems in learning to code. Those tools have some expectations with them, though, notably that you're willing to read a lot, install things, and be very comfortable with things simply not working the first time.

Most of programming is debugging and very carefully following instructions.

The following steps will be a little tricky, and need to be done in order, or you will not be able to submit your labs for this class. If you do not submit your labs, you can't get credit for them - but more importantly, you won't be able to practice your debugging before your group project.

Let's do this together.

  • Your node installation must be above version 13.2.0. 
  • This means installing the Current and not LTS version of Node from https://nodejs.org/en/download/
  •  To check your version of Node.js, open a Terminal in VSCode, and type the command node --version
    Press Enter.

Part 1 - Getting Set Up

  • Sign up for Github.com if you have not yet done so.
    • Use a professional name that you would be happy to submit for a job interview
  • Find and install Github Desktop if you have not yet done so.
  • Open https://pages.github.com (Links to an external site.) in a new tab.
  • Follow the instructions exactly on Github Pages to create a Pages-connected repository on your Github account
    • If you're sure you did it right and you still get a 404 Not Found, try adding /index.html to your URL, like this: https://username.github.io/index.html
    • Github often takes time to update or change.
  • Clone or open your new Pages repository in Github Desktop.
  • Clone a copy of the course repository to your Github Desktop.
    • Cloning the Class Repository 
    • The course repo contains testing tools to help you get better at code, faster.

Part 2 - Installing Your Lab Space

  • This requires some comfort navigating directories in your computer.
  • Open two Explorer or Finder windows
  • Copy the contents of the course repository (Ideally to be found at Documents/Github/INST377-FA2020) into your Github Pages folder/lab repository (Ideally to be found at Documents/Github/yourUsername.github.io)
  • You will need to edit the name of the following files:
    • gitignore -> .gitignore
    • env -> .env
    • eslint.cjs -> .eslint.cjs
    • this is because "dot files" are ignored by your file system.
  • Open your lab repository in VSCode using Github Desktop
    • You should see something like this image in your VSCode client

  • Important:
    • Edit your .env file to contain your name and email
    • ie: NAME="your name" becomes NAME="Alex Leitch"
  • Open a terminal window in your VSCode editor by clicking "Terminal > New Terminal" in your menus.
  • Type npm install. The results should look sort of like this:

  • Everything you need to test your work should now exist on your computer.
  • If you have trouble with any stage of this, please ask for help in the class Discord! Your peers probably ran into the same issue, there are a lot of you.
  • Remember to screencap your problem when you're asking for help. Clear explanations get you better answers.
